GM Commands

Here is a not quite definitive listing of the GM Commands and their functions. There may be some commands which do not appear in this text. All of these commands are available for use by Full GM's and a lesser number of them will function for the Assistant GM's.

ALL GM commands are preceded by an @ (at) sign and most may be abbreviated to their first 3 characters.

- Mugg L. Wump

@ACTIVATE [monster name]

    This command 'turns on' a monster which is currently 'turned off' in the room. It will begin to attack, move, etc. at this point.
Ex: @ACT goblin, @ACT 2 skeleton

@ADDITEM [item archetype]

    This command places the item with the specified archetype into the GM's current room. NOTE: The item is immediately visible to ANYONE within the same room.
Ex: @ADD 45, this would create a sword (archetype 45) in the current room.

@ADDITEM [item archetype] [record number]

    This command clones the item with the specified archetype into the GM's current room. NOTE: The item is immediately visible to ANYONE within the same room.

@ADJECTIVE

    Displays a list of all Adjective numbers and names within the system. Has the extra feature of disconnecting you while sending them. Other commands are better for this function.

@ANNOUNCE [1,2,3,10,11] [text msg]

    Announces the specified string everyone inthe specified range.

@ANSWER

    If there is an ASSIST pending in your game this will take you to the location of the next one which is pending.

@BANK [player #]

    Displays the contents of the defined players bank account.

@BET

    Unlike the @BID command you do not need a preset location to activate this command. Betting opens in the GM's current location when this command is toggled on and closes when toggled off.

@BID [amount]

    Opens bidding at a specified 'bidding location' at the amount provided. An amount of -1 closes the bidding and deducts the amount from the highest bidder. See @BIDLOC.

@BIDLOC [0-5]

    Defines a room as a 'bidding location'. See @BID.

@CLOSE [item name]

    Closes the first such item in the current room.

@DELETE [item name]

    Removes the first item of this type within the GM's current room or inventory. Be sure to check which item you are deleting before you do it, since it can get scary otherwise.
Ex: @DEL sword (Will delete the first sword item within the room/inventory), @DEL green flask (Will delete that specific item in room/inventory)

@ECHOPLR [player #] [text]

    Displays the text to the specified player and only the specified player no matter where they are in the game.

@EDITITEM [item name]

    Edits the first item of this type within the GM's current room or inventory. It will show you the item you are deleting immediately after you start to, so it is easy to see if it is the correct one. The following options display one at a time after invoking this command on an item.
Ex: @EDIT sword

@EDN [Database Record #]

    Functions exactly like @EDITITEM but will effect the item of the defined Database Record. In other words, you can change an item anywhere in the game by looking up it's Record #.

@EDPLR [player #]

    This will display a list of the attributes that can be edited on a player. It is best to learn this one on yourself first. ;)

@EDPLR [player #] [field] [value]

    This will allow you to set certain player values. To get a list of available choices for field, replace it with a ?. It is best to learn this one on yourself first. ;)

@EDSKILL [player #] [skill #] [new rank]

    Changes the defined players matching skill # to the newly defined level.
Ex: @EDS 20 16 15 ---] Give player 20 15 ranks in Skill #16.

@ENTRY [text]

    Makes the text display to any players in a room you enter with @GO/@GOP if you are visible.

@EVIL [player #]

    Grants 1 'evil' point to the defined player. This should be used with caution as it influences the player's aura and should have some basis in their current Roleplaying.
Ex: @EVIL 20 --] Gives one 'evil' point to player 20.

@EXAMINE [item archetype]

    Shows some basic information about the defined item.
Ex: @EXA 45 --] Basic information about a sword.

@EXCLUDE [player #] [text]

    Displays the text to all players in the same room with the 'excluded' player, and obviously not to that person.

@EXIT [text]

    Makes the text display to any players in a room you leave with @GO/@GOP if you are visible.

@EXP [player #] [experience amount]

    Gives the defined amount of EXPERIENCE points to the effected player. Changes their current level and build points accordingly. Currently does *not* allow for negative values.
Ex: @EXP 20 10000 --] Grants 10000 experience points to player 20.

@FIND [item archetype]

    Lists all items with the defined archetype currently in the game no matter where they are located (players,rooms,bank,etc.). Also provides the Database Record # and item description.
Ex: @FIND 45 --] Will display all 'sword's of that type in the game.

@GENMON [monster #]

    Generates the specified monster in the GM's current room. When the monster is generated it is 'sedated' and therefore inactive. Use of the @ACT command will turn it on.
Ex: @GEN 1 --] Would generate a skeleton in the current room.

@GET [Database Record #]

    Takes the defined item from anywhere in the game and places it into the GM's inventory. NOTE : Be EXTREMELY careful with this command, although it should only accept numerics it will take other values and often you will find yourself holding a door or tunnel from someplace. Avoid using it unless you are sure of the Database number and really need to.
Ex: No example provided, try not to use it too much. ;)

@GLOSS [item name]

    Will provide a list of all items,nouns,adjectives and monster adjectives with the specified name (abbreviations can be used). Useful for finding items and such that you are looking for.

@GM

    Turns on GM Mode.gm hat on, monsters will not attack, you will see all whispers in the room, and if on WHO list you will be listed on duty.

@GO [Room #]

    Teleports the GM to the specified Room #.
Ex: @GO 206 --] Takes the GM to Bazaar Center.

@GOOD [player #]

    See @EVIL.

@GOP [Player #]

    Teleport the GM to the specified player's location.
Ex: @GOP 20 --] Takes GM to player 20.

@GRANTSP [player #] [spell #]

    Adds the specified spell to the players spell list regardless of whether they can cast it.

@HEAL [player #]

    Fully heals the specified player number, even from death.

@HIDE

    Removes the GM's name from the WHO listing.

@INVIS

    Turns the GM invisible to players. They will still be able to see a number of the GM's actions, such as verbs. You can also still be heard if you talk.

@KILL [player #]

    Opposite of @HEAL, simply kills the specified player.

@LINEx [player#] [text]

    This is used to alter the three appearance lines of the player (@LINE1, @LINE2 and @LINE3). Use x to reset the line to its starting value, or -none- to display nothing on that line.

@LIST

    Lists all nouns in the game, will disconnect you. Useless.

@LOCK [item name]

    Will lock the first such item in the room.

@LSKILLS

    Lists all skills available within the game.

@LWHO

    Gives a detailed listing of players in the game, including full names and locations. See @WHO.

@MLIST

    Lists all monsters currently defined in the game.

@MSG

    A toggle command which turns on/off [Script:] and [REPORTS:].

@NUM [player name]

    Will display the player # of for matching names, works with abbreviations.

@PEEK [variable]

    Displays the current value of the defined variable.

@PINV [player #]

    Displays the current inventory of the defined player with Database Record #'s.

@PSI [player #] [psi #]

    Gives the specified player the defined Psionic power regardless of whether they can use it or not.

@OPEN [item name]

    Opens the first such item in the current room.

@QSTAT [player #]

    Gives a brief overview of the defined players name, level, and various other things.

@QUEUE [record #] [count]

    queues item in room for treasure on a searched monster. Count is how many monsters until item is found. The item itself must be left on the floor of the room. See @UNQUEUE.

@RDATA [room number]

    Displays information about the defined room including number of players within it and items with Database Record #'s.

@REGION [Region #]

    Displays a listing of the special attributes of the defined region. (Regions)

@REGISTER [player #] [team]

    This is how you place people on a team for use with the @BET command. Your options are -1 to cancel the registry, 0 to not place them on any team, 1 to place them on Team A or 2

@RESTORE [NPC #]

    This will change your character into whatever is stored in the specified NPC slot. All items, abilities, etc. that you currently have will be replaced by the new ones, so if you want to keep your current character be sure to @SAVE it first. *NOTE* : Do NOT use this command unless you have been given a slot as there are a few mundane characters saves in slots and you will become ordinary.

@RFLAG

    Turns off GM mode. gm hat off, monsters will attack, you will not see whispers, and if on WHO list you will be listed as a player.

@SAVE [NPC #]

    This saves all information concerning your current character to the specified slot. It will overwrite whatever is in that slot so be sure to only use it if you have been assigned a number by Damnthing (holder of the sacred numbers.)

@SCRIPT

    Begins the process where the GM can upload a script been instructed as to how.

@SEDATE [monster name]

    'Turns off' the first such monster in the room. See @ACT/@GENMON/@ZAPMON.

@SET [variable] [val]

    Changes the value of a system variable

@SND [text]

    Displays the complete text message to everyone within the GM's current room. Using a \ will act as a Carriage Return to split up the lines.

@SPAWN [monster #]

    Creates an active monster.

@SPEECH [player #] [text]

    Changes the way the defined player talks.Note: The text of the speech type can only 12 characters.
Ex: @SPEECH 1 burble --] If the player spoke you would see. Mugglewump burbles, "Hey, what happened?"

@SPELL [player #]

    Gives detailed information on the specified players known spells and psionic powers.

@STATUS [player #]

    Gives detailed information on the specified player including stats, level, skills, and specializations.

@SUMMON [monster #]

    Summons an active monster to your command.

@TIME [new value]

    Changes the current 'world' time. Values are 0-23 where 0 is Midnight and 12 is Noon.

@TRACE

    Toggles on/off the GM trace mode. Little is known of how this functions so it is of limited use.

@TREASURE [treasure level]

    Generates the appropriate amount/type of treasure in the GM's current room.

@UNHIDE

    Places the GM's name on the WHO listing.

@UNLOCK [item name]

    See @LOCK.

@UNQUEUE [record #] [count]

    removes an item from the monster queues. It cannot be simply deleted. See @QUEUE.

@VISIBLE

Makes the GM visible to players, etc. See @INVIS.

@WHISPER [player #] [text]

    Whispers the text to the defined player.
The player will see a message like follows. 
          [Mugglewump]: Hey, what are you doing silly person?

@WHO

    Lists all players and monsters currently within the game complete with player #'s.

@YANK [player #]

    Takes the defined player from whereever they are to the GM's current location.

@ZAPMON [monster name]

    Removes the first of the specified monster from the room/game.
