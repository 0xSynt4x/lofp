General Scripting

Conditional statements

There are a number of control commands available within the scripting languae, which will be discussed later in this section. Most of these controls are best used within conditional statements. The script language offers a variety of IF type expressions for allowing you to program what happens when people do certain things or certain conditions are met.

As a matter of being orderly it is usual for conditional statements to be placed at the end of the item room block, after you have placed all of the basic definitions.

Basic outline of a conditional statement: 

IF [arguments]
.
.    [control commands or additional conditions]
.
ENDIF

The following is a list of the available conditional statements.

  • IFITEM
  • IFTOUCH
  • IFVERB
  • IFVERB2
  • IFPREVERB
  • IFPREVERB2
  • IFVAR
  • IFNOITEM
  • IFFULLDESC
  • IFENTRY
  • IFSEEK
  • IFSAY

IFIN [item tag #] [var]

This if statement is true if the item archetype specified by 'var' is located inside the item specified by the item tag number. If it is found, the "current item" is set to the first such item located.

IFCARRY [var] [plr #]

This if statement is true if the item archetype specified by 'var' is located in the current player's inventory. If it is found, the "current item" is set to the first such item located.

Variables and Arithmetic

Variables may be defined to keep track of numeric values. To define a variable, simply place VARIABLE [varaible name] on a line by itself anywhere in your script.

You can perform arithmetic functions on your variables or base conditional statements upon them. All of them assume the format: 

[arithmetic function] [destination variable name] [source variable or value]
The available arithmetic funstions are: 
ADD    | Addition
DIV    | Division
EQUAL  | Equal to
MOD    | Modulus division
MULT   | Multiplication
SUB    | Subtraction
Example: 
VARIABLE TESTVAR
VARIABLE VAR2
EQUAL TESTVAR 3
EQUAL VAR2 7
ADD VAR2 3
MUL TESTVAR VAR2
The above would define two variables, "Testvar and Var2." Testvare is set equal to 3 and Var2 equal to 7. The number 3 is then added to Var2 making it 10. Testvar is then multiplied by Var2.

SKILLCHECK [skill#] [+/- tochance] [var]

will check (skill# * 5%) +/- tochance + base chance = %chance
%chance - roll = Success or Failure
If %chance is + Sucess
If %chance is - Failure
Base chance, 0 ranks of skill#, 25 % chance

SHOWROOM [x]

Echos the room description of room x to the current player as if they had typed "look" in that room.

ZAPMON

Deletes the monster from the room as well as anything it was carrying.

RECALC

This causes the player's offensive and defensive statistics to be recalculated based on their current environment. This is mainly important for making sure peoples OM/DM are correct after passing through planar portals.

MLIST [group #] [monster #] [ratio] [qty]
