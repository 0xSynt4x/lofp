GM Manual - Game Engine

Architecture

The game consists of several major components: the databases which hold all of the data for rooms, items, monsters and players, the script language which interprets the scripts defining rooms, items and monsters, and the command interpreter which processes command verbs.

The script language is used to define rooms, items and monsters, and to handle verbs for rooms and items. Most commands can have their behavior changed by 'trapping' the verb inside a script. Whatever isn't scripted is handled by default code in the engine, which is difficult to change. The default behavior can be changed by the script for the room the player is in, or the script for the item, or a special script which works for all rooms.
